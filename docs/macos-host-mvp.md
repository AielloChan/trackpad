# macOS Host MVP

The macOS host MVP receives platform-neutral input events and maps them to local macOS input commands.

## Capabilities

- Reads Accessibility permission status.
- Requests Accessibility permission.
- Injects pointer movement through `CGEvent`.
- Maps pointer button, tap, and scroll protocol events to macOS input commands.
- Maps consecutive tap events to CoreGraphics mouse click state for native double-click behavior.
- Receives newline-delimited JSON `SessionFrame` frames over TCP.
- Requires a valid pairing hello before processing input frames.
- Advertises the TCP server with Bonjour.
- Exposes server status in the native macOS app.
- Writes persistent diagnostic logs for startup, shutdown, listener state, connection state, pairing results, input event summaries, mapped commands, and send errors.

## LAN Service

Default TCP port:

```text
44787
```

Bonjour service type:

```text
_trackpad-host._tcp
```

The first MVP uses newline-delimited JSON. Each session frame is one JSON object followed by `\n`.

Every client must send `clientHello` before sending input:

```json
{"clientHello":{"protocolVersion":1,"deviceId":"trackpad-cli","deviceName":"Trackpad CLI","pairingCode":"123456"}}
```

Input is then sent as an `input` session frame:

```json
{"input":{"version":1,"sequenceNumber":1,"timestampNanos":1000,"kind":{"pointerMove":{"dx":80,"dy":0}}}}
```

The exact JSON shape is generated by Swift `Codable` and should be treated as a development format. A compact cross-platform schema can replace it after the event model stabilizes.

## CLI

From:

```bash
cd apps/macos/TrackpadHost
```

Check permission:

```bash
swift run TrackpadHost status
```

Print the host log path:

```bash
swift run TrackpadHost log-path
```

Start the host server:

```bash
swift run TrackpadHost serve 123456
```

Send a local sample pointer move event:

```bash
swift run TrackpadHost send-sample-event 123456
```

Local debug input actions:

```bash
swift run TrackpadHost move-test
swift run TrackpadHost left-click-test
swift run TrackpadHost right-click-test
swift run TrackpadHost scroll-test
```

Only run click and scroll tests in a safe empty UI area.

## Native App

Open:

```text
apps/macos/TrackpadHostApp/TrackpadHostApp.xcodeproj
```

Run the `TrackpadHostApp` scheme. The app shows:

- Accessibility permission state.
- Server state.
- Current pairing code.
- Port.
- Active connection count.
- Authorized connection count.
- Handled event count.
- Host log file path.
- Last error, if any.

## Logs

The default host log file is:

```text
~/Library/Logs/Trackpad/host.log
```

The file logger rotates `host.log` to `host.log.1` after it reaches roughly 2 MB. Logs are intentionally plain text so bug reports can be diagnosed by reading this file directly.

## Verified Locally

The LAN loop was verified by running `serve 123456`, then `send-sample-event` with an invalid and valid pairing code.

Invalid pairing code:

```text
before 769.8046875 -629.38671875
after  769.8046875 -629.38671875
handled 0
last error invalid pairing code
```

Valid pairing code:

```text
before 769.8046875 -629.38671875
after  849.8046875 -629.38671875
handled 1
```

Observed host logs:

```text
Host error: invalid pairing code
Host status: running, port: 44787, connections: 1, authorized: 1, handled: 1
Host status: running, port: 44787, connections: 0, authorized: 0, handled: 1
```

This verifies:

```text
TCP client -> LAN server -> session hello gate -> JSON Lines decoder -> HostEventProcessor -> MacInputInjector -> CGEvent pointer movement
```
